## README: TakeHomeProject - High Income Prediction and Segmentation

This guide provides instructions for setting up the environment, executing the data science notebooks, and understanding the output artifacts of the `TakeHomeProject`.

### Project Overview

This project analyzes census data to predict income levels ($\le$ 50K vs. $>$ 50K) and create customer segments for targeted marketing strategies. The analysis includes data preprocessing, exploratory analysis, predictive modeling, and customer segmentation.

---

### Prerequisites

You must have **Python 3.10+** installed on your system

### Setup

1.  **Clone the Repository (if applicable):**
    ```bash
    git clone <url>
    cd TakeHomeProject
    ```

2.  **Install Required Libraries:**
    Install all necessary Python packages using the provided `requirements.txt` file.
    ```bash
    pip install -r requirements.txt
    ```

### Execution Steps

The  analysis is performed by executing the four Jupyter notebooks in sequential order.

1.  **Execute Notebooks in below order:**

    | Notebook | Purpose | Outputs |
    | :--- | :--- | :--- |
    | **1\_Data\_Preprocessing.ipynb** | Prepares raw data by cleaning, imputing, and transforming features | `data/processed_data.parquet` (Cleaned data), `~$bivariate_report.xlsx` (Bivariate analysis) |
    | **2\_Data\_Exploration\_and\_Visualization.ipynb** | Performs EDA, data quality checks, and initial feature analysis | `artifacts/data_quality/` (e.g., `summary_report.xlsx`, plots) |
    | **3\_Data\_Modeling.ipynb** | Trains, evaluates, and compares multiple classification models for high income prediction | `artifacts/model_objects/` (Trained models), `artifacts/model_plots/model_comparison.png` |
    | **4\_Data\_Segmentation.ipynb** | Builds a shallow decision tree to identify and profile distinct customer segments | `artifacts/model_plots/segment_profile.png`, `artifacts/model_plots/segments_summary.csv` |

    > **Note:** **Run all cells** in each notebook from top to bottom before moving to the next one. The subsequent notebook relies on the outputs generated by the previous one.
